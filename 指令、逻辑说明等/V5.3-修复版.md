# 分区分层动态热度评分方案 (V5.3 – 修复版)

**版本说明**\
本版本在 V5.2 基础上，根据逻辑漏洞分析报告进行了以下关键修复：

1. **修复流程循环依赖**：将趋势判定前置至评分之前，使自适应风控系数可正确设定。
2. **明确物理特征和值取整规则**：采用四舍五入取整。
3. **增强动态关联规则挖掘稳定性**：提高支持度阈值至 ≥3，并加入最小提升度要求。
4. **补全冷号集中回补规则**：完整描述四阶段判定与不重开铁律。
5. **极端拐点替换并列处理**：明确按热度分降序，同分时按号码从小到大选择替换对象。
6. **协调高热拥堵限制与自适应风控**：在强热/强冷趋势下，允许顶峰+降权区总数 ≤4 时全部可参与主推。

其余规则（永久分区、双向动态跳过/回归、冷号缩权等）同 V5.2。\
版本号提升至 **V5.3**。

***

## 一、基础体系

### 1.1 永久分区

| 类别     | 号码                         |
| :----- | :------------------------- |
| **热号** | 01, 05, 06, 07, 12         |
| **冷号** | 02, 03, 04, 08, 09, 10, 11 |

### 1.2 挂靠联动 → 动态关联规则挖掘

**原规则**：固定配对（如02↔05）已废除。\
**新规则**：每期基于**最近50期**数据，使用Apriori算法挖掘强关联对（前项→后项），要求：

- 支持度 ≥ 3（即规则至少出现3次）
- 置信度 ≥ 60%
- 提升度 (lift) > 1.5（排除负相关）

若存在规则 `(X→Y)`，则上期开出 X 时，本期 Y 获得 +5分（挂靠加分）。\
取置信度最高的前3条规则应用。若当期无活跃规则，此项为0。\
初始数据不足50期时，可沿用固定配对 `02↔05, 08↔04, 11↔07, 10↔02→04, 03→01` 作为过渡。

***

## 二、核心评分公式

```
最终热度分 = (基础热度分 + 热度变化率 + 遗漏值分 + 动态挂靠加分 + 二阶链加分 + 物理特征加分)
             × 连出修正系数 × 自适应风控系数
```

### 2.1 基础热度分（多窗口加权）

- 近36期出现次数 × 1
- 近24期出现次数 × 2
- 近12期出现次数 × 3
- 近7期出现次数 × 7
- 近3期出现次数 × 8\
  以上求和。

### 2.2 热度变化率

```
热度变化率 = (近3期出现次数 × 10) - (前3期（再往前3期）出现次数 × 5)
```

### 2.3 遗漏值分（倒U曲线）

| 遗漏值范围   | 分值                |
| :------ | :---------------- |
| ≤10期    | 遗漏值 × 1.3         |
| 10\~16期 | 13（封顶）            |
| 16\~22期 | 13 - (遗漏值-16) × 2 |
| >22期    | 0                 |

### 2.4 动态挂靠加分

- 每期根据最新关联规则（基于最近50期）计算，对满足前项条件的号码给予对应后项 +5分。
- 若当前无活跃规则，则此项为0。

### 2.5 二阶马尔可夫链加分（滑动窗口200期）

- 维护一个 **200期滑动窗口** 的转移计数矩阵 `Count[12][12][12]`。
- 每期新数据加入时，移除最旧一期对应的三元组计数（减1），然后新三元组计数加1。
- 基于当前计数矩阵计算转移概率：\
  `P(x) = Count[prev2][prev1][x] / Σ_all Count[prev2][prev1][x]`

加分规则：

| 转移概率         | 加分  |
| :----------- | :-- |
| ≥ 30%        | +10 |
| 20%\~30%     | +7  |
| 10%\~20%     | +4  |
| 5%\~10%      | +2  |
| 0\~5%        | +1  |
| 状态未出现（总次数为0） | 0   |

### 2.6 物理特征加分（新增）

基于下期号码的奇偶和和值进行预测。

- **奇偶预测**：\
  计算最近20期开奖号码的奇偶比例。若奇数占比 > 60%，则预测下期为偶数；反之为奇数。\
  所有符合预测奇偶性的号码获得 **+3分**。
- **和值预测**：\
  计算最近20期开奖号码的平均值，**四舍五入取整**得 Avg。预测下期号码在 `[Avg-2, Avg+2]` 范围内（包含边界）。\
  所有落在此范围内的号码获得 **+2分**。

**每个号码物理特征加分上限**：5分（奇偶3分 + 和值2分，若同时符合则加满）。

### 2.7 连出修正系数

| 当前连出次数  | 系数   |
| :------ | :--- |
| 0连（未连开） | 1.0  |
| 1连      | 0.85 |
| 2连      | 0.6  |
| ≥3连     | 0.3  |

### 2.8 自适应风控系数

12期频次五区制的风控系数根据**当前趋势强度**动态调整。趋势判定需在评分前完成（见流程）。

| 12期频次分区  | 强热/强冷趋势 | 震荡趋势 |
| :------- | :------ | :--- |
| 顶峰区（≥4次） | 0.5     | 0.3  |
| 降权区（=3次） | 0.7     | 0.5  |
| 轮动区（=2次） | 1.0     | 1.0  |
| 等待区（=1次） | 1.0     | 1.0  |
| 静默区（=0次） | 1.0     | 1.0  |

**解权机制**：顶峰/降权区号码，若近11期出现次数 ≤2，则自动恢复为轮动区系数（1.0）。

## 三、12期滑动窗口频次五区制

| 区间名     | 12期次数 | 风控系数（见2.8） | 推荐池权限    | 动态跳过联动         |
| :------ | :---- | :--------- | :------- | :------------- |
| **顶峰区** | ≥4    | 自适应        | 移出主推，仅备选 | 冷号触发硬跳过→转热观察   |
| **降权区** | =3    | 自适应        | 移出主推，可备选 | 冷号触发硬跳过→转热观察   |
| **轮动区** | =2    | 1.0        | 正常参与主推   | 冷号可软跳过→转热观察    |
| **等待区** | =1    | 1.0        | 正常候选     | 热号可软跳过→转冷观察    |
| **静默区** | =0    | 1.0        | 正常候选     | 热号触发硬/软跳过→转冷观察 |

**高热拥堵限制（修正版）**：

- 若顶峰区+降权区号码总数 > 4，仅允许其中近11期次数由3降为2的号码参与主推，其余限备选。
- 若总数 ≤4，则全部可参与主推（受各自风控系数限制）。
- 在强热/强冷趋势下，该限制自动调整为总数 ≤4 时全部可主推（已含在上条中），不再进一步收紧。

## 四、双向动态冷热跳过/回归机制

### 4.1 热号 → 阶段转冷（跳过）

| 条件                                | 跳过类型           | 观察期 |
| :-------------------------------- | :------------- | :-- |
| 近12期0次 + 近24期≤1次 + 近36期≤3次        | **硬跳过**，即时移出热池 | 3期  |
| 近12期0次 + 近24期≤2次 + 近36期≤5次，且近7期全0 | **软跳过**        | 3期  |

- 观察期内若再开出，撤销跳过，恢复热身份。
- 期满未出，正式归入冷池。

### 4.2 冷号 → 阶段转热（跳过）

| 条件                              | 跳过类型           | 观察期 |
| :------------------------------ | :------------- | :-- |
| 近12期≥3次（即进入降权/顶峰区）              | **硬跳过**，即时移出冷池 | 2期  |
| 近12期=2次（轮动区）+ 近24期≥4次 + 近36期≥6次 | **软跳过**        | 2期  |

- 观察期内若再开出，撤销跳过，恢复冷身份。
- 期满未出，正式归入热池，取消冷号缩权。

### 4.3 回归（无需观察，直接切换）

- **转冷热号回归热池**：不再满足转冷条件（如近12期出现≥1次）自动回归。
- **转热冷号回归冷池**：近12期0次且近24期≤2次且近36期≤4次，自动回归冷池，恢复缩权。

**动态身份优先于固定身份**，推荐策略中的有效热池/冷池指动态身份后的池。

## 五、其他规则

### 5.1 冷号缩权

固定冷号（且未处于“冷→热观察”状态）在近2期内开出，最终热度分 ×0.1。\
转热冷号不受此限。

### 5.2 极冷警戒（遗漏 ≥36期）

- 若趋势判定为**震荡**，则强制将该号码加入备选（替换原备选末位）；多个则取遗漏值最大的两号。
- 其他趋势下，仅输出预警信息，不干预推荐池。

### 5.3 行情优先级

冷号集中回补 > 极端拐点 > 常规趋势。

### 5.4 冷号集中回补（四阶段+不重开铁律）

当某固定冷号（或已进入冷→热观察期的号码）满足以下条件时，触发冷号集中回补行情：

**阶段一（启动）**：近7期出现2次，且其关联热号（依据动态关联规则中的最高置信度规则）在近3期出现至少1次。\
**阶段二（加速）**：近7期出现3次以上，且关联热号在近3期出现2次以上。\
**阶段三（高潮）**：近7期连续出现2次以上，且关联热号同步连续出现。\
**阶段四（尾声）**：近7期出现次数开始下降（如连续2期未出），或关联热号断开。

**不重开铁律**：同一号码在一次冷号集中回补行情结束后（最后一次出现后的5期内），若再次触发启动条件，忽略本次启动，间隔至少5期后才可重新认定。

**执行**：

- 阶段一：在震荡趋势下，将该冷号提升至主推第一位（替代原热池①）。
- 阶段二及以上：在所有趋势下，直接将该冷号提升至主推第一位，并相应调整备选。
- 若同时有多个冷号触发，取近7期出现次数最多的优先。

### 5.5 趋势判定（近3期，基于动态身份后的实际热冷比）

- **强热**：热号数量 ≥ 2
- **强冷**：冷号数量 ≥ 2
- **震荡**：热冷各1\~2个

### 5.6 极端拐点（连续3期单边即触发）

- 当近3期开奖全部为**热号**（动态热池）或全部为**冷号**（动态冷池）时，判定为极端拐点趋势。
- **主推**中强制加入1个反向区间（与当前趋势相反的区间）中热度分最高的号码。
- **备选**中加入2个反向区间号码（若备选已满则替换其中顺位最低的）。
- 执行时，主推仍保持4码：由反向号码替换原主推中热度分最低的一个（若原主推已有反向号码则免）。若热度分最低有多个并列，则按号码从小到大选取（小号优先替换）。

## 六、推荐策略

### 6.1 强热趋势

- 主推：有效热池前4名（按最终热度分排序）
- 备选：热池第5 + 有效冷池第1

### 6.2 强冷趋势

- 主推：有效冷池前3名 + 总热度顺位第4（跳过已选号码）
- 备选：冷池第4 + 有效热池第1

### 6.3 震荡趋势

- 主推：热池① + 热池② + 冷池① + 变盘06
- 备选：冷池② + 热池③
- 若06受风控限制（顶峰/降权或转冷），顺延热池④。

### 6.4 极端拐点

- 在以上对应策略基础上，按5.6规则修改主推和备选。

***

## 七、完整计算流程（每期执行顺序）

1. **更新滚动窗口**：36/24/12/7/3期出现次数，12期频次分区。
2. **更新关联规则池**：基于最近50期数据挖掘强关联对，更新挂靠联动表。
3. **更新二阶马尔可夫链滑动矩阵**：移除最旧一期三元组计数，加入最新一期计数。
4. **五区制与解权**：确定每个号码的推荐池权限（主推/备选）及自适应风控系数的基础值。
5. **动态身份**：运行双向跳过/回归逻辑，构建有效热池/冷池。
6. **趋势判定**（此步提前）：基于近3期实际开奖号码和动态身份，判定趋势类型（强热/强冷/震荡），并检查是否存在极端拐点（连续3期单边）。
7. **确定自适应风控系数**：根据步骤6判定的趋势，结合步骤4的分区，为每个号码设定风控系数。
8. **计算物理特征加分**：基于最近20期统计奇偶和和值，对符合预测的号码加分。
9. **计算每个号码的最终热度分**：
   - 基础分 + 变化率 + 遗漏分 + 动态挂靠分 + 二阶链分 + 物理特征分
   - 乘连出修正系数（基于动态身份）
   - 乘自适应风控系数（步骤7确定的）
10. **按最终热度分排序**。
11. **检查冷号集中回补触发条件**，若有则按5.4规则调整主推/备选。
12. **按对应策略（含极端拐点修正）** 选取主推4码、备选2码。
13. **极冷预警**（遗漏≥36期输出 ⚠️ 提醒）。
14. **输出推荐结果**。
