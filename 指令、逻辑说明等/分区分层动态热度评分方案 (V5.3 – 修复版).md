# 分区分层动态热度评分系统 V5.3 – 完整实施方案

## 一、方案概述

本系统基于12个开奖号码（1\~12）的历史数据，通过多维度评分、动态身份识别、关联规则、马尔可夫链、物理特征预测等算法，自动生成每期的推荐主推4码和备选2码，并适配强热、强冷、震荡、极端拐点、冷号集中回补等特殊行情。

**核心特点**：

- 评分公式由7个因子组成，经连出修正和自适应风控调整
- 动态冷热跳过/回归机制，避免固化身份
- 行情优先级：冷号集中回补 > 极端拐点 > 常规趋势
- 自修复：修复了V5.2中的6个逻辑漏洞

## 三、核心算法详解

### 3.1 基础数据窗口

| 窗口   | 权重 | 对应函数                  |
| :--- | :- | :-------------------- |
| 近36期 | ×1 | `computeWindowFreq()` |
| 近24期 | ×2 | 同上                    |
| 近12期 | ×3 | 同上                    |
| 近7期  | ×7 | 同上                    |
| 近3期  | ×8 | 同上                    |

**备注**：不足对应期数时，取全部可用期数。

### 3.2 热度变化率

`changeRate = (近3期出现次数 × 10) - (再往前3期出现次数 × 5)`

### 3.3 遗漏值分（倒U曲线）

| 遗漏值范围  | 分值                   |
| :----- | :------------------- |
| ≤10    | `miss × 1.3`         |
| 10\~16 | 13（封顶）               |
| 16\~22 | `13 - (miss-16) × 2` |
| >22    | 0                    |

### 3.4 动态挂靠加分

- 基于最近50期挖掘关联规则（Apriori）：支持度≥3，置信度≥60%，提升度>1.5
- 取前3条最高置信度规则，若前项出现在上期，则后项获得 +5分
- 若 history.length < 50，使用过渡规则（固定配对）

### 3.5 二阶马尔可夫链加分

- 维护200期滑动计数矩阵 `Count[12][12][12]`
- 增量更新：移除最旧三元组，加入最新三元组
- 转移概率 `P(x|prev2,prev1)`，对应加分：
  - ≥30% → +10
  - 20%\~30% → +7
  - 10%\~20% → +4
  - 5%\~10% → +2
  - 0\~5% → +1
  - 未出现 → 0

### 3.6 物理特征加分（上限5分）

- **奇偶预测**：近20期奇数占比>60%则预测偶数，否则预测奇数 → 符合 +3分
- **和值预测**：近20期平均值四舍五入取整Avg，号码在 \[Avg-2, Avg+2] 内 → +2分
- 两者可叠加，但总加分 ≤5分

### 3.7 连出修正系数

| 当前连出次数 | 修正系数 |
| :----- | :--- |
| 0连（未连） | 1.0  |
| 1连     | 0.85 |
| 2连     | 0.6  |
| ≥3连    | 0.3  |

### 3.8 自适应风控系数（含解权机制）

- 根据预判趋势（评分前完成趋势判定）和12期频次分区，动态设定：
  - 强热/强冷趋势：顶峰区0.5，降权区0.7
  - 震荡趋势：顶峰区0.3，降权区0.5
  - 轮动/等待/静默区：1.0
- **解权机制**：若顶峰区/降权区号码近11期出现次数 ≤2，则恢复为1.0

### 3.9 五区制与高热拥堵限制

| 12期频次 | 分区 | 初始风控系数（见3.8） |
| :---- | :- | :----------- |
| ≥4    | 顶峰 | 动态           |
| =3    | 降权 | 动态           |
| =2    | 轮动 | 1.0          |
| =1    | 等待 | 1.0          |
| =0    | 静默 | 1.0          |

**高热拥堵限制**：

- 若顶峰区+降权区总数 >4，仅允许其中近11期次数由3降为2的号码参与主推
- 若总数 ≤4，则全部可参与主推（受各自风控系数限制）
- 在强热/强冷趋势下，该限制自动调整为总数 ≤4 时全部可主推

### 3.10 双向动态跳过/回归状态机

**热→冷跳过**：

| 条件                                 | 跳过类型 | 观察期 |
| :--------------------------------- | :--- | :-- |
| 近12期0次 + 近24期≤1次 + 近36期≤3次         | 硬跳过  | 3期  |
| 近12期0次 + 近24期≤2次 + 近36期≤5次 + 近7期全0 | 软跳过  | 3期  |

**冷→热跳过**：

| 条件                          | 跳过类型 | 观察期 |
| :-------------------------- | :--- | :-- |
| 近12期≥3次（降权/顶峰）              | 硬跳过  | 2期  |
| 近12期=2次 + 近24期≥4次 + 近36期≥6次 | 软跳过  | 2期  |

**回归条件**：

- 转冷热号回归热池：近12期出现≥1次
- 转热冷号回归冷池：近12期0次 + 近24期≤2次 + 近36期≤4次

**状态机执行**：

- 每次新一期加入时，调用 `updateDynamicState`，更新每个号码的状态，包括观察期减少、跳出状态、撤销跳过等。

### 3.11 冷号缩权

固定冷号（2,3,4,8,9,10,11）且**未处于冷→热观察状态**，若在**近2期**内开出，则最终热度分乘以0.1。

### 3.12 趋势判定（评分前完成）

基于近3期实际开奖号码的动态身份（热/冷）计数：

- 热号 ≥2 → 强热
- 冷号 ≥2 → 强冷
- 否则 → 震荡

同时检测**极端拐点**：连续3期全部为热号或全部为冷号。

### 3.13 冷号集中回补（四阶段+不重开铁律）

触发条件：

- **阶段一**：该冷号近7期出现2次，且关联热号（最高置信度规则）近3期出现≥1次，且趋势为震荡
- **阶段二**：近7期≥3次，关联热号近3期≥2次
- **阶段三**：近7期连续出现≥2次，且关联热号同步连续出现
- **阶段四**：近7期出现次数下降（如连续2期未出），或关联热号断开

**不重开铁律**：同一号码在最后一次出现后的5期内，忽略再次触发启动。

**执行**：

- 阶段一（仅震荡趋势）：将该冷号提升至主推第一位
- 阶段二及以上（所有趋势）：直接提升至主推第一位
- 多个号码取近7期出现次数最多的优先

### 3.14 极端拐点修正

当检测到极端拐点时：

- 主推中强制加入1个反向区间（与当前趋势相反）中**热度分最高的号码**
- 替换原主推中热度分**最低**的号码（若并列，取号码最小的替换）
- 备选中加入2个反向区间号码（若已满则替换末位）

### 3.15 推荐策略

| 趋势   | 主推4码                                | 备选2码          |
| :--- | :---------------------------------- | :------------ |
| 强热   | 有效热池前4名                             | 热池第5 + 有效冷池第1 |
| 强冷   | 有效冷池前3名 + 总排名第4（跳过已选）               | 冷池第4 + 有效热池第1 |
| 震荡   | 热池① + 热池② + 冷池① + 变盘06（若06受限则顺延热池④） | 冷池② + 热池③     |
| 极端拐点 | 在以上基础上应用3.14修正                      | 同上            |

### 3.16 极冷预警（遗漏≥36期）

- 震荡趋势下：强制将该号码加入备选（替换原备选末位；多个取遗漏最大的两个）
- 其他趋势下：仅输出警告信息

### 3.17 完整计算流程（每期）

1. 更新窗口频次（36/24/12/7/3期）
2. 更新关联规则池（最近50期）
3. 更新马尔可夫链滑动矩阵（200期窗口）
4. 五区分类与解权（确定基础风控参数）
5. 运行动态跳过/回归状态机 → 得到有效热池/冷池
6. **趋势判定**（基于近3期动态身份）→ 趋势、极端拐点标志
7. 确定每个号码的自适应风控系数（结合趋势、分区、解权）
8. 计算物理特征加分（近20期）
9. 计算每个号码的最终热度分（各因子求和 → 乘连出修正 → 乘风控系数 → 冷号缩权）
10. 按最终分降序排序（并列时按号码升序）
11. 检测冷号集中回补（含不重开铁律检查）
12. 应用高热拥堵限制
13. 按对应策略（含极端拐点修正）生成主推4码、备选2码
14. 生成极冷预警
15. 输出完整结果
