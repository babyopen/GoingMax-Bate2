# 分区分层动态热度评分方案 V5.3（修复版）

<br />

***

## 一、基础体系

### 1.1 永久分区（实战评分主依据）

| 类别     | 号码                         |
| :----- | :------------------------- |
| **热号** | 01, 05, 06, 07, 12         |
| **冷号** | 02, 03, 04, 08, 09, 10, 11 |

### 1.2 挂靠联动 → 动态关联规则挖掘

废除固定配对，改为每期基于最近 **50 期** 数据挖掘强关联对（前项 → 后项），要求：

- 支持度 ≥ 3
- 置信度 ≥ 60%
- 提升度 (lift) > 1.5

若存在规则 `(X → Y)`，则上期开出 X 时，本期 Y 获得 **+5 分**（挂靠加分）。取置信度最高的前 3 条应用。初始数据不足 50 期时，可沿用过渡固定配对：\
`02↔05, 08↔04, 11↔07, 10↔02→04, 03→01`。

***

## 二、核心评分公式

```
最终热度分 = (基础热度分 + 热度变化率 + 遗漏值分 + 动态挂靠加分 + 二阶链加分 + 物理特征加分)
             × 连出修正系数 × 自适应风控系数
```

### 2.1 基础热度分（多窗口加权）

- 近 36 期出现次数 × 1
- 近 24 期出现次数 × 2
- 近 12 期出现次数 × 3
- 近 7 期出现次数 × 7
- 近 3 期出现次数 × 8\
  以上求和。

### 2.2 热度变化率

```
(近3期出现次数 × 10) - (前3期出现次数 × 5)
```

### 2.3 遗漏值分（倒U曲线）

| 遗漏值范围      | 分值                  |
| :--------- | :------------------ |
| ≤ 10 期     | 遗漏值 × 1.3           |
| 10 \~ 16 期 | 13（封顶）              |
| 16 \~ 22 期 | 13 - (遗漏值 - 16) × 2 |
| > 22 期     | 0                   |

### 2.4 动态挂靠加分

每期根据最新关联规则（基于最近 50 期）计算，对满足前项条件的号码给予对应后项 +5 分。若无活跃规则，此项为 0。

### 2.5 二阶马尔可夫链加分（滑动窗口 200 期）

维护一个 **200 期滑动窗口** 的转移计数矩阵 `Count[12][12][12]`。\
每期新数据加入时，移除最旧一期对应的三元组计数（减 1），然后新三元组计数加 1。\
基于当前矩阵计算转移概率：

```
P(x) = Count[prev2][prev1][x] / Σ_all Count[prev2][prev1][x]
```

加分规则：

| 转移概率       | 加分  |
| :--------- | :-- |
| ≥ 30%      | +10 |
| 20% \~ 30% | +7  |
| 10% \~ 20% | +4  |
| 5% \~ 10%  | +2  |
| 0 \~ 5%    | +1  |
| 状态未出现      | 0   |

### 2.6 物理特征加分（新增）

- **奇偶预测**：计算最近 20 期奇偶比例，若奇数占比 > 60% 则预测下期为偶数，反之为奇数。所有符合预测奇偶性的号码 +3 分。
- **和值预测**：最近 20 期号码平均值 **四舍五入取整** 得 Avg，预测范围 `[Avg-2, Avg+2]`（含边界）。落在范围内的号码 +2 分。

**单个号码上限**：5 分（奇偶 3 + 和值 2，同时符合则加满）。

### 2.7 连出修正系数

| 当前连出次数   | 系数   |
| :------- | :--- |
| 0 连（未连开） | 1.0  |
| 1 连      | 0.85 |
| 2 连      | 0.6  |
| ≥3 连     | 0.3  |

### 2.8 自适应风控系数

由 12 期频次分区和当前趋势共同决定。

| 12 期频次分区  | 强热/强冷趋势 | 震荡趋势 |
| :-------- | :------ | :--- |
| 顶峰区（≥4 次） | 0.5     | 0.3  |
| 降权区（=3 次） | 0.7     | 0.5  |
| 轮动区（=2 次） | 1.0     | 1.0  |
| 等待区（=1 次） | 1.0     | 1.0  |
| 静默区（=0 次） | 1.0     | 1.0  |

**解权机制**：顶峰/降权区号码若近 11 期出现次数 ≤ 2，自动恢复为轮动区系数（1.0）。

## 三、12 期滑动窗口频次五区制

| 区间名     | 12 期次数 | 风控系数（自适应） | 推荐池权限 | 动态跳过联动         |
| :------ | :----- | :-------- | :---- | :------------- |
| **顶峰区** | ≥4     | 见 2.8     | 仅备选   | 冷号触发硬跳过→转热观察   |
| **降权区** | =3     | 见 2.8     | 仅备选   | 冷号触发硬跳过→转热观察   |
| **轮动区** | =2     | 1.0       | 正常主推  | 冷号可软跳过→转热观察    |
| **等待区** | =1     | 1.0       | 正常候选  | 热号可软跳过→转冷观察    |
| **静默区** | =0     | 1.0       | 正常候选  | 热号触发硬/软跳过→转冷观察 |

**高热拥堵限制（修正版）**：

- 若顶峰区+降权区号码总数 > 4，仅允许其中近 11 期次数由 3 降为 2 的号码参与主推，其余限备选。
- 若总数 ≤ 4，全部可参与主推（受各自风控系数限制）。
- 在强热/强冷趋势下，该限制自动保持不动（已满足 ≤4 可全部主推）。

## 四、双向动态冷热跳过/回归机制

### 4.1 热号 → 阶段转冷（跳过）

| 条件                                | 跳过类型           | 观察期 |
| :-------------------------------- | :------------- | :-- |
| 近12期0次 + 近24期≤1次 + 近36期≤3次        | **硬跳过**，即时移出热池 | 3期  |
| 近12期0次 + 近24期≤2次 + 近36期≤5次，且近7期全0 | **软跳过**        | 3期  |

- 观察期内若再开出，撤销跳过，恢复热身份。
- 期满未出，正式归入冷池。

### 4.2 冷号 → 阶段转热（跳过）

| 条件                              | 跳过类型           | 观察期 |
| :------------------------------ | :------------- | :-- |
| 近12期≥3次（即进入降权/顶峰区）              | **硬跳过**，即时移出冷池 | 2期  |
| 近12期=2次（轮动区）+ 近24期≥4次 + 近36期≥6次 | **软跳过**        | 2期  |

- 观察期内若再开出，撤销跳过，恢复冷身份。
- 期满未出，正式归入热池，取消冷号缩权。

### 4.3 回归（无需观察，直接切换）

- **转冷热号回归热池**：不再满足转冷条件（如近12期出现≥1次）自动回归。
- **转热冷号回归冷池**：近12期0次且近24期≤2次且近36期≤4次，自动回归冷池，恢复缩权。

**动态身份优先于固定身份**，推荐策略中的“有效热池/冷池”指动态身份后的池。

## 五、其他规则

### 5.1 冷号缩权

固定冷号（且未处于“冷→热观察”状态）在近 2 期内开出，最终热度分 × 0.1。\
转热冷号不受此限。

### 5.2 极冷警戒（遗漏 ≥ 36 期）

- 若趋势为**震荡**，强制将该号码加入备选（替换原备选末位）；多个则取遗漏值最大的两个。
- 其他趋势仅输出预警信息，不干预推荐池。

### 5.3 行情优先级

冷号集中回补 > 极端拐点 > 常规趋势。

### 5.4 冷号集中回补（四阶段 + 不重开铁律）

当某固定冷号（或已进入冷→热观察期的号码）满足以下条件时触发：

**阶段一（启动）**：近 7 期出现 2 次，且其关联热号（依据动态关联规则中的最高置信度规则）在近 3 期出现至少 1 次。\
**阶段二（加速）**：近 7 期出现 3 次以上，且关联热号在近 3 期出现 2 次以上。\
**阶段三（高潮）**：近 7 期连续出现 2 次以上，且关联热号同步连续出现。\
**阶段四（尾声）**：近 7 期出现次数开始下降（如连续 2 期未出），或关联热号断开。

**不重开铁律**：同一号码在一次冷号集中回补行情结束后（最后一次出现后的 5 期内），若再次触发启动条件，则忽略本次启动，间隔至少 5 期后才可重新认定。

**执行**：

- 阶段一：在震荡趋势下，将该冷号提升至主推第一位（替代原热池①）。
- 阶段二及以上：在所有趋势下，直接将该冷号提升至主推第一位，并相应调整备选。
- 若同时有多个冷号触发，取近 7 期出现次数最多的优先。

### 5.5 趋势判定（近 3 期，基于动态身份后的实际热冷比）

- **强热**：热号数量 ≥ 2
- **强冷**：冷号数量 ≥ 2
- **震荡**：热冷各 1 \~ 2 个

### 5.6 极端拐点（连续 3 期单边即触发）

- 当近 3 期全部为热号或全部为冷号时，判定为极端拐点。
- **主推**中强制加入 1 个反向区间（与当前趋势相反）中热度分最高的号码。
- **备选**中加入 2 个反向区间号码（若备选已满则替换其中顺位最低的）。
- 执行时，主推保持 4 码：由反向号码替换原主推中热度分最低的一个（若原主推已有反向号码则免）。若热度分最低有并列，按号码从小到大选取。

## 六、推荐策略（基于动态有效池）

### 6.1 强热趋势

- 主推：有效热池前 4 名（按最终热度分排序）
- 备选：热池第 5 + 有效冷池第 1

### 6.2 强冷趋势

- 主推：有效冷池前 3 名 + 总热度顺位第 4（跳过已选号码）
- 备选：冷池第 4 + 有效热池第 1

### 6.3 震荡趋势

- 主推：热池① + 热池② + 冷池① + 变盘 06
- 备选：冷池② + 热池③
- 若 06 受风控限制（顶峰/降权或转冷），顺延热池④。

### 6.4 极端拐点

- 在以上对应策略基础上，按 5.6 规则修改主推和备选。

***

## 七、完整计算流程（每期执行顺序）

1. **更新滚动窗口**：36/24/12/7/3 期出现次数，12 期频次分区。
2. **更新关联规则池**：基于最近 50 期数据挖掘强关联对，更新挂靠联动表。
3. **更新二阶马尔可夫链滑动矩阵**：移除最旧一期三元组计数，加入最新一期计数。
4. **五区制与解权**：确定每个号码的推荐池权限（主推/备选）及自适应风控系数的基础值。
5. **动态身份**：运行双向跳过/回归逻辑，构建有效热池/冷池。
6. **趋势判定**（此步提前）：基于近 3 期实际开奖号码和动态身份，判定趋势类型（强热/强冷/震荡），并检查极端拐点。
7. **确定自适应风控系数**：根据步骤 6 判定的趋势，结合步骤 4 的分区，为每个号码设定风控系数。
8. **计算物理特征加分**：基于最近 20 期奇偶和和值，对符合预测的号码加分。
9. **计算最终热度分**：
   - 基础分 + 变化率 + 遗漏分 + 动态挂靠分 + 二阶链分 + 物理特征分
   - 乘连出修正系数（基于动态身份）
   - 乘自适应风控系数（步骤 7 确定的）
10. **排序**（按最终热度分降序）。
11. **检查冷号集中回补触发条件**，若有则按 5.4 规则调整主推/备选。
12. **按对应策略（含极端拐点修正）** 选取主推 4 码、备选 2 码。
13. **极冷预警**（遗漏 ≥ 36 期输出 ⚠️）。
14. **输出推荐结果**。

***

*本方案自 V5.3 起最终定型，所有规则闭环，维护时仅需按周期优化参数（如滑动窗口大小、关联规则阈值、风控系数等）。*
